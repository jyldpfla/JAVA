● flash (FlashAttributeController.java)
: 1번의 redirect에 대해 값을 유지해줄 수 있는 attribute를 spring에서 제공

● Interceptor ( filter랑 비슷한 역할)
: handler, controller 구현되기 전 후로 동작
(MyController.java, myview.jsp, MyInterceptor.java, WebConfig.java)

▶ override method
// preHandle : controller가 사용자 요청을 처리하기 전
// postHandle : controller가 사용자의 요청을 처리하고 난 뒤, view가 만들어지기 전
// afterCompletion : view가 다 만들어지고 난 뒤 (dispatcher serlvet이 맞는 view를 찾아서 넘겨준 뒤)

● error
(ErrorHandler.java, errorpage.jsp, MyController.java
: 예외적인 상황이 발생했을 때 사용자는 발생한 error에 대해 화면으로 보아야 함 + 개발자는 예외에 대해 기록이 남겨있어져야 함
- 흐름 전역에서 error가 발생했을 때마다 handling하는 error handler 필요
▶ @ControllerAdvice : controller 영역에서 예외 핸들링 등 controller 전역에서 반복적으로 일어나는 일을 관리할 수 있는 component
